Rita-DSL Demo

This is a demo of open source project rita-dsl which is powered by another awesome project spaCy

  • info_outline Short syntax guide

    MY_RULE->MARK("MY_LABEL") Left side represents a rule, right side - how to mark matched text (giving it a label)
    Multiple rules can be combined as follows: {RULE1, RULE2, ..., RULEn}->MARK("MY_LABEL")

    Basic rules:
    • WORD - matches any word. Adding concrete value, like: WORD("car") - matches that concrete word
    • LEMMA - matches by lemma. In short, it means that any form of word should match. Eg.: LEMMA("like") would match: like, likes, liked
    • IN_LIST - matches any word in given list
    • ENTITY - matches by entity label. These come from spaCy. Usual ones are: PERSON, ORG (organization), GPE (geopolitical entity). Refer to spaCy.io for more information, these come from NER (named entity recognition) which could have different labels depending on language
    Most of the rules can have modifiers:
    • * - 0 to infinity times.
    • + - 1 to infinity times.
    • ? - Optional value. Can match once or don't match at all.

    For a full syntax guide, refer to our Documentation

  • http API

    For large research or production systems, on-premises solution is only way to go. However, it might be too inconvenient and time consuming for some hobby/small solutions.

    We do provide an API which allows to do full analysis via simple HTTP calls (which is being used by frontend). To access it, you do need account with API token created and request limit applied. Currently it is done manually on request.
